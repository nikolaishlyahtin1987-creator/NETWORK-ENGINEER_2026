**Таблица 1**

|IP-адрес узла:|	192.168.200.139

|Найти:|Ответ|
|:---|---:|
|Количество бит подсети|3	
|Количество созданных подсетей|	8
|Количество бит узлов в подсети|5	
|Количество узлов в подсети|30	
|Сетевой адрес этой подсети|192.168.200.128	
|IPv4-адрес первого узла в этой подсети|192.168.200.129	
|IPv4-адрес последнего узла в этой подсети|192.168.200.158    	
|Широковещательный IPv4-адрес в этой подсети|192.168.200.159	

Рассмотрим детальное решение каждого пункта.         
**1. Количество бит подсети**        
&nbsp;&nbsp;&nbsp;&nbsp;- **Исходная маска** 255.255.255.0       
Переводим в двоичный вид: 11111111.11111111.11111111.00000000            
Получаем 24 единицы, значит, **префикс /24**     
&nbsp;&nbsp;&nbsp;&nbsp;- **Новая маска** 255.255.255.224    
В доичном виде: 11111111.11111111.11111111.11100000      
27 единиц, значит, **префикс /27**    
&nbsp;&nbsp;&nbsp;&nbsp;- **Количество бит подсети**     
Вычисляем: 27-24 = 3 бита подсети

**Ответ: 3 бита подсети.**

**2. Посчитаем количество созданных подсетей**      
Количество возможных подсетей:    
  $2^s=2^3=8$       
  где S - количество дополнительных бит
 
 **Ответ: 8 подсетей**

**3. Количество бит узлов в подсети**      
&nbsp;&nbsp;&nbsp;&nbsp;- Новая маска подести 255.255.255.224     
В двоином виде 11111111.1111111.11111111.11100000  (префикс /27)     
&nbsp;&nbsp;&nbsp;&nbsp;- Количество бит узлов      
Общая длина IP-адреса — **32 бита**.
Биты узлов = 32 − (префикс)     
32-27=5 бит     

**Ответ: 5 бит**

**4. Количество узлов в подсети**  
&nbsp;&nbsp;&nbsp;&nbsp;- Количество узлов в подсети вычисляется по формуле:  
$2^n-2$ гле n - количество бит узлов.         
Минус 2 — потому что один адрес зарезервирован под номер сети, другой под широковещательный адрес.       
Подставляем:
$2^5-2$=32-2=30  

**Ответ: 30 узлов в каждой подести**   

**5. Сетевой адрес этой подсети**  
&nbsp;&nbsp;&nbsp;&nbsp;Для маски /27 шаг подсети = $2^5$ = 32 адреса     
&nbsp;&nbsp;&nbsp;&nbsp;Подсети для сети 192.168.200.0/24 с маской /27:      
0, 32, 64, 96, **128**, 160, 192, 224      
Узел 139 попадает в диапазон 128–159     

**Ответ: сетевой адрес этой подсети 192.168.200.128**

**6. IPv4-адрес первого узла в этой подсети**      
&nbsp;&nbsp;&nbsp;&nbsp;- Сетевой адрес подсети 192.168.200.128    
&nbsp;&nbsp;&nbsp;&nbsp;- Первый узел в подсети = сетевой адрес + 1     
&nbsp;&nbsp;&nbsp;&nbsp;- 192.168.200.128 + 1 = 192.168.200.129     

**Ответ: 192.168.200.129**     

**7. IPv4-адрес последнего узла в этой подсети**    
&nbsp;&nbsp;&nbsp;&nbsp; -Последний узел = широковещательный адрес − 1      
192.168.200.159−1 = 192.168.200.158  

**Ответ: 192.168.200.158**     

**8. Широковещательный IPv4-адрес в этой подсети**     
&nbsp;&nbsp;&nbsp;&nbsp; -Широковещательный адрес = сетевой адрес + $2^n-1$ где n количество бит узлов     
&nbsp;&nbsp;&nbsp;&nbsp;- Бит узлов = 32 − 27 = 5 бит    
&nbsp;&nbsp;&nbsp;&nbsp;- $2^5$ = 32 адреса в подсети     
&nbsp;&nbsp;&nbsp;&nbsp;-  Широковещательный = сетевой адрес + 31   
&nbsp;&nbsp;&nbsp;&nbsp; - 192.168.200.128 + 31 = 192.168.200.159    

**Ответ: 192.168.200.159**

**7. IPv4-адрес последнего узла в этой подсети**  
 &nbsp;&nbsp;&nbsp;&nbsp; - Найдём широковещательный адрес для подсети 192.135.250.176/29   
&nbsp;&nbsp;&nbsp;&nbsp; Сетевой адрес: 192.135.250.176       
&nbsp;&nbsp;&nbsp;&nbsp; В четвёртом байте: 176 = 10110000

&nbsp;&nbsp;&nbsp;&nbsp; Устанавливаем все биты узлов в 1:  
&nbsp;&nbsp;&nbsp;&nbsp; 10110111 = 183         
&nbsp;&nbsp;&nbsp;&nbsp;Широковещательный адрес: 192.135.250.183

### **Вопрос для повторения**    
**Почему маска подсети так важна при анализе IPv4-адреса?** 

Маска подсети важна при анализе IPv4-адреса, потому что без неё IP-адрес сам по себе не определяет границы сети и узла.
